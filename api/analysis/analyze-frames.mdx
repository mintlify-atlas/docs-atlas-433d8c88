## Overview

This endpoint analyzes a video post for deepfake content by processing extracted frames through the primary detection model. It calculates confidence scores, fake/real frame counts, and updates the post with comprehensive analysis results.

## Authentication

This endpoint requires authentication via JWT token.

<Note>
Ensure the Authorization header includes a valid Bearer token.
</Note>

## Path Parameters

<ParamField path="postId" type="string" required>
  The MongoDB ObjectId of the post to analyze
</ParamField>

## Response

Returns the updated post object with analysis results embedded in the `deepfake_analysis` field.

<Expandable title="Response Schema">
  <ResponseField name="_id" type="string">
    Post ID
  </ResponseField>

  <ResponseField name="title" type="string">
    Post title
  </ResponseField>

  <ResponseField name="content" type="string">
    Post content
  </ResponseField>

  <ResponseField name="media_url" type="string">

          <ResponseField name="total_frames" type="number">
            Total number of frames analyzed
          </ResponseField>

          <ResponseField name="real_frames" type="number">
            Count of frames classified as real

          <ResponseField name="fake_frames" type="number">
            Count of frames classified as fake

  </ResponseField>

  <ResponseField name="updatedAt" type="string">
    Timestamp of last update

</Expandable>

## Implementation Details

<Info>
This endpoint communicates with a Flask server running on `localhost:5000` to perform the actual frame analysis using the primary deepfake detection model.
</Info>

The analysis process:
1. Locates the frames directory based on the video filename
2. Sends frame directory path to Flask analysis service
3. Receives per-frame analysis results
4. Calculates summary statistics (fake/real counts, confidence percentage)
5. Updates the post with complete analysis results
6. Sets `analysis_status` to "completed"
