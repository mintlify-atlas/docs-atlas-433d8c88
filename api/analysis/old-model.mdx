## Overview

This endpoint analyzes a video post using the legacy (old) deepfake detection model. It extracts frames from the video, processes them through the old model, and stores results in a separate collection for comparison purposes.

<Note>
This endpoint first extracts frames if needed, then analyzes them using the legacy model running on port 5001. Results are stored separately from the primary analysis.
</Note>

## Authentication

This endpoint requires authentication via JWT token.

## Path Parameters

<ParamField path="postId" type="string" required>
  The MongoDB ObjectId of the post to analyze
</ParamField>

## Response

Returns an `OldModelAnalysis` object containing the legacy model's analysis results.

<Expandable title="Response Schema">
  <ResponseField name="_id" type="string">
    Analysis document ID
  </ResponseField>

  <ResponseField name="post_id" type="string">
    Reference to the analyzed post
  </ResponseField>

  <ResponseField name="is_fake" type="boolean">
    Overall determination if the video is fake according to the old model
  </ResponseField>

  <ResponseField name="confidence" type="number">
    Overall confidence score from the old model
  </ResponseField>

  <ResponseField name="frames_analysis" type="array">
    Array of per-frame analysis results
    
    <Expandable title="frame object">
      <ResponseField name="frame" type="string">
        Frame filename
      </ResponseField>

      <ResponseField name="confidence" type="number">
        Confidence score for this frame (0-1)
      </ResponseField>

      <ResponseField name="is_fake" type="boolean">
        Whether this frame is detected as fake
      </ResponseField>

      <ResponseField name="frame_path" type="string">
        Path to the frame image file
      </ResponseField>
    </Expandable>
  </ResponseField>

  <ResponseField name="summary" type="object">
    Analysis summary statistics
    
    <Expandable title="summary properties">
      <ResponseField name="status" type="string">
        Overall verdict: "FAKE" or "REAL"
      </ResponseField>

      <ResponseField name="confidence_percentage" type="number">
        Confidence as percentage (0-100), calculated as (fake_frames / total_frames) * 100
      </ResponseField>

      <ResponseField name="total_frames" type="number">
        Total number of frames analyzed
      </ResponseField>

      <ResponseField name="real_frames" type="number">
        Count of frames classified as real
      </ResponseField>

      <ResponseField name="fake_frames" type="number">
        Count of frames classified as fake
      </ResponseField>
    </Expandable>
  </ResponseField>

  <ResponseField name="createdAt" type="string">
    Timestamp when analysis was created
  </ResponseField>

  <ResponseField name="updatedAt" type="string">
    Timestamp of last update
  </ResponseField>
</Expandable>

## Behavior Notes

<Info>
**Idempotent Operation**: If an analysis already exists for the given post, the existing analysis is returned immediately without re-processing.
</Info>

## Implementation Details

The analysis process follows these steps:

1. **Check Existing Analysis**: Returns cached results if already analyzed
2. **Extract Frames**: Calls Flask service on port 5000 to extract video frames
3. **Analyze with Old Model**: Sends frames to legacy Flask service on port 5001
4. **Calculate Summary**: Computes statistics from frame-level results
5. **Store Results**: Saves analysis in `OldModelAnalysis` collection
6. **Return Analysis**: Sends complete results to client

<Note>
The old model uses a separate Flask server instance (port 5001) to avoid conflicts with the primary model.
</Note>
