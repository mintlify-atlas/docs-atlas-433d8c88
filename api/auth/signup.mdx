---
title: 'Sign Up'
api: 'POST /signup'
---

Create a new user account in the IPD system.

## Request Body

<ParamField body="username" type="string" required>
  User's email address. Must be a valid email format.
</ParamField>

<ParamField body="firstName" type="string" required>
  User's first name.
</ParamField>

<ParamField body="lastName" type="string" required>
  User's last name.
</ParamField>

<ParamField body="password" type="string" required>
  User's password for authentication.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message confirming user creation.
</ResponseField>

<ResponseField name="token" type="string">
  JWT authentication token for the newly created user. Contains the `userId` claim.
</ResponseField>

## Code Examples

<CodeGroup>

```bash curl
curl -X POST https://your-api-domain.com/signup \
  -H "Content-Type: application/json" \
  -d '{
    "username": "user@example.com",
    "firstName": "John",
    "lastName": "Doe",
    "password": "securePassword123"
  }'
```

```javascript JavaScript
const response = await fetch('https://your-api-domain.com/signup', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    username: 'user@example.com',
    firstName: 'John',
    lastName: 'Doe',
    password: 'securePassword123'
  })
});

const data = await response.json();
console.log(data.token); // JWT token
```

</CodeGroup>

## Error Responses

### Validation Error (411)

Returned when request body fails validation or email is already registered.

```json
{
  "message": "Email already taken / Incorrect inputs"
}
```

<Note>
The `username` field must be a valid email address. The Zod validation schema enforces email format validation.
</Note>

<Warning>
Passwords are currently stored in plain text in the database. This is not recommended for production use. Consider implementing password hashing with bcrypt or similar libraries.
</Warning>

## Validation Rules

- **username**: Must be a valid email address (validated by Zod's `.email()` method)
- **firstName**: Required string field
- **lastName**: Required string field  
- **password**: Required string field (no minimum length enforced)
- Email uniqueness is checked - duplicate emails return error code 411

## Implementation Details

The endpoint uses Zod schema validation defined in backend/index.js:34-39:

```javascript
const signupBody = zod.object({
    username: zod.string().email(),
    firstName: zod.string(),
    lastName: zod.string(),
    password: zod.string()
})
```

The JWT token is signed with the `JWT_SECRET` environment variable and contains the `userId` claim.
