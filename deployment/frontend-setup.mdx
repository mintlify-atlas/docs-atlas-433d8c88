## Overview

The IPD frontend is built with React 18, Vite, TypeScript, and Tailwind CSS, providing a modern and responsive user interface for the Intelligent Post Detector.

  <Step title="Configure Environment Variables">
    Create a `.env` file in the `frontend` directory:
    
    ```bash
    touch .env
    ```
    
    Add the following configuration:
    
    ```bash .env
    # Backend API URL
    VITE_API_URL=http://localhost:3000
    
    # Flask ML Servers
    VITE_FLASK_SERVER_1_URL=http://localhost:5000
    VITE_FLASK_SERVER_2_URL=http://localhost:5001
    
    # Google Maps API (Optional - only if using location features)
    VITE_GOOGLE_MAPS_API_KEY=your_google_maps_api_key
    
    # App Configuration
    VITE_APP_NAME=IPD - Intelligent Post Detector
    VITE_APP_VERSION=1.0.0
    ```
    
    <Note>
      All environment variables in Vite must be prefixed with `VITE_` to be exposed to the client.
    </Note>
    
    <Warning>
      The `.env` file is excluded from version control. For production, set these variables in your hosting platform's environment configuration.
    </Warning>
  </Step>

  <Step title="Update API Endpoints (if needed)">
    If your backend runs on a different host/port, update the environment variables:
    
    <CodeGroup>
    ```bash Local Development
    VITE_API_URL=http://localhost:3000
    VITE_FLASK_SERVER_1_URL=http://localhost:5000
    VITE_FLASK_SERVER_2_URL=http://localhost:5001
    ```
    
    ```bash Production
    VITE_API_URL=https://api.yourdomain.com
    VITE_FLASK_SERVER_1_URL=https://ml1.yourdomain.com
    VITE_FLASK_SERVER_2_URL=https://ml2.yourdomain.com
    ```
    

<Tabs>
  <Tab title="Development Mode">
    Start the Vite development server with hot module replacement:
    
    ```bash

    ```


</Tabs>

## API Integration

The frontend communicates with the backend via Axios:

<CodeGroup>
```typescript Authentication
// Example: Sign in
import axios from 'axios';

const API_URL = import.meta.env.VITE_API_URL;

const signin = async (username: string, password: string) => {
  const response = await axios.post(`${API_URL}/signin`, {
    username,
    password
  });
  return response.data;
};
```

```typescript Video Analysis
// Example: Analyze video
const analyzeVideo = async (videoFile: File) => {
  const formData = new FormData();
  formData.append('video', videoFile);
  
  const response = await axios.post(
    `${import.meta.env.VITE_FLASK_SERVER_1_URL}/analyze`,
    formData,
    {
      headers: { 'Content-Type': 'multipart/form-data' }
    }
  );
  return response.data;
};
```
</CodeGroup>

## Deployment

<Tabs>
  <Tab title="Netlify">
    1. Build the project:
       ```bash
       npm run build
       ```
    
    2. Deploy to Netlify:
       ```bash
       npm install -g netlify-cli
       netlify deploy --prod --dir=dist
       ```
    
    3. Set environment variables in Netlify dashboard:
       - Go to Site settings → Environment variables
       - Add all `VITE_*` variables
  </Tab>
  
  <Tab title="Vercel">
    1. Install Vercel CLI:
       ```bash
       npm install -g vercel
       ```
    
    2. Deploy:
       ```bash
       vercel --prod
       ```
    
    3. Set environment variables:
       ```bash
       vercel env add VITE_API_URL
       vercel env add VITE_FLASK_SERVER_1_URL
       vercel env add VITE_FLASK_SERVER_2_URL
       ```
  </Tab>
  
  <Tab title="Static Hosting">
    After building (`npm run build`):
    
    1. Upload the `dist/` directory to your hosting provider
    2. Configure the server to:
       - Serve `index.html` for all routes (SPA fallback)
       - Set proper MIME types
       - Enable gzip/brotli compression
    
    Example Nginx configuration:
    ```nginx
    server {
      listen 80;
      server_name yourdomain.com;
      root /path/to/dist;
      index index.html;
      
      location / {
        try_files $uri $uri/ /index.html;
      }
      
      # Cache static assets
      location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
      }
    }
    ```
  </Tab>
  
  <Tab title="Docker">
    Create a `Dockerfile` in the frontend directory:
    
    ```dockerfile
    FROM node:18-alpine AS builder
    WORKDIR /app
    COPY package*.json ./
    RUN npm ci
    COPY . .
    RUN npm run build
    
    FROM nginx:alpine
    COPY --from=builder /app/dist /usr/share/nginx/html
    COPY nginx.conf /etc/nginx/conf.d/default.conf
    EXPOSE 80
    CMD ["nginx", "-g", "daemon off;"]
    ```
    
    Build and run:
    ```bash
    docker build -t ipd-frontend .
    docker run -p 80:80 ipd-frontend
    ```
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 5173 Already in Use">
    Change the dev server port in `vite.config.ts`:
    ```typescript
    export default defineConfig({
      server: {
        port: 5174, // or any available port
      },
    });
    ```
  </Accordion>

  <Accordion title="Module Not Found Errors">
    Clear cache and reinstall:
    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="CORS Errors">
    Ensure backend has CORS enabled for your frontend URL:
    ```javascript
    // backend/index.js
    app.use(cors({
      origin: 'http://localhost:5173'
    }));
    ```
  </Accordion>

  <Accordion title="Environment Variables Not Working">
    - Ensure variables are prefixed with `VITE_`
    - Restart the dev server after changing `.env`
    - Check you're using `import.meta.env.VITE_*` not `process.env`
  </Accordion>

  <Accordion title="TypeScript Errors">
    Run type checking:
    ```bash
    npx tsc --noEmit
    ```
    
    Install missing type definitions:
    ```bash
    npm install --save-dev @types/node @types/react @types/react-dom
    ```
  </Accordion>

  <Accordion title="Build Fails">
    Common fixes:
    ```bash
    # Clear Vite cache
    rm -rf node_modules/.vite
    
    # Clear TypeScript cache
    rm -rf node_modules/.cache
    
    # Rebuild
    npm run build
    ```
  </Accordion>
</AccordionGroup>
