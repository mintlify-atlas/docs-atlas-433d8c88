## Overview

The IPD application uses two Flask servers running machine learning models for deepfake detection:

1. **Flask Server 1 (Port 5000)**: Vision Transformer (ViT) model
2. **Flask Server 2 (Port 5001)**: CNN+LSTM model

Both servers work together to analyze video content and detect potential deepfakes.

## Architecture

<Steps>
  <Step title="Video Upload">
    User uploads video through the frontend
  </Step>
  <Step title="Frame Extraction">
    Backend extracts frames from video using FFmpeg
  </Step>
  <Step title="Model Analysis">
    Both ML models analyze extracted frames independently
  </Step>
  <Step title="Result Aggregation">
    Results are combined to determine if video is fake or real
  </Step>
</Steps>

## Flask Server 1 Configuration (ViT)

### Model Architecture

The Vision Transformer model is configured with the following parameters:

```python
# backend/flask_server/app.py:42-52
model = ViT(
    image_size=224,      # Input image size
    patch_size=32,       # Patch size for transformer
    num_classes=1,       # Binary classification
    dim=1024,           # Embedding dimension
    depth=6,            # Number of transformer layers
    heads=16,           # Number of attention heads
    mlp_dim=2048,       # MLP hidden dimension
    dropout=0.1,        # Dropout rate
    emb_dropout=0.1     # Embedding dropout rate
)
```

### Directory Structure

```bash
flask_server/
├── app.py              # Main Flask application
├── models/             # Model weights directory
│   └── as_model_0.837.pt
├── uploads/            # Temporary video uploads
└── frames/            # Extracted video frames
```

## Frame Extraction Process

Both servers use OpenCV to extract frames from videos:

```python
# Extracts every 20th frame
frames_num = 20
if count % frames_num == 0:
    frame_path = os.path.join(frames_dir, f"frame{count}.jpg")
    cv2.imwrite(frame_path, image)
```

This reduces processing time while maintaining analysis accuracy.

## Confidence Scoring

### How It Works

<Steps>
  <Step title="Individual Frame Analysis">
    Each frame receives a confidence score (0.0 to 1.0):
    - **High score (greater than 0.5)**: Frame appears real
    - **Low score (less than 0.5)**: Frame appears fake
  </Step>

  <Step title="Frame Classification">
    ```python
    is_fake = confidence < 0.5
    ```
  </Step>

  <Step title="Video-Level Decision">
    ```python
    fake_frames = sum(1 for frame in frames_analysis if frame['is_fake'])
    real_frames = num_frames - fake_frames
    is_fake = fake_frames > real_frames  # Majority vote
    ```
  </Step>

  <Step title="Average Confidence">
    ```python
    average_confidence = total_confidence / num_frames
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Model File Not Found">
    **Error:**
    ```
    FileNotFoundError: Model file not found at models/[model_name]
    ```
    
    **Solution:**
    ```bash
    # Check if model file exists
    ls -lh backend/flask_server/models/as_model_0.837.pt
    ls -lh backend/flask_server_2/models/deepfake_detection_model.h5
    
    # Create models directory if missing
    mkdir -p backend/flask_server/models
    mkdir -p backend/flask_server_2/models
    
    # Copy model files to correct location
    ```
  </Accordion>

  <Accordion title="Port Already in Use">
    **Error:**
    ```
    OSError: [Errno 98] Address already in use
    ```
    
    **Solution:**
    ```bash
    # Find process using the port
    lsof -i :5000
    lsof -i :5001
    
    # Kill the process
    kill -9 <PID>
    
    # Or use a different port
    # In app.py, change:
    app.run(host='0.0.0.0', port=5002, debug=True)
    ```
  </Accordion>

  <Accordion title="CUDA/GPU Errors">
    **Error:**
    ```
    RuntimeError: CUDA out of memory
    ```
    
    **Solution:**
    ```python
    # Force CPU usage (in app.py)
    import torch
    torch.set_default_device('cpu')
    
    # Or for TensorFlow:
    import os
    os.environ['CUDA_VISIBLE_DEVICES'] = '-1'
    ```
  </Accordion>

  <Accordion title="OpenCV Import Error">
    **Error:**
    ```
    ImportError: libGL.so.1: cannot open shared object file
    ```
    
    **Solution:**
    ```bash
    # Ubuntu/Debian
    sudo apt install -y libsm6 libxext6 libxrender-dev libgomp1 libglib2.0-0
    
    # Or use headless OpenCV
    pip uninstall opencv-python
    pip install opencv-python-headless
    ```
  </Accordion>

  <Accordion title="Model Loading Errors">
    **PyTorch model errors:**
    ```bash
    # Ensure PyTorch version matches training version
    pip install torch==2.1.0 torchvision==0.16.0
    ```
    
    **TensorFlow model errors:**
    ```bash
    # Try loading without compilation
    model = tf.keras.models.load_model(model_path, compile=False)
    ```
  </Accordion>

  <Accordion title="Memory Issues">
    **Solution:**
    ```python
    # Reduce batch processing
    # Process fewer frames at once
    
    # Clear frames after analysis
    import shutil
    shutil.rmtree(frames_dir)
    ```
  </Accordion>
</AccordionGroup>

## Production Deployment

<Warning>
  The default Flask development server is not suitable for production. Use a production WSGI server.
</Warning>

### Using Gunicorn

```bash
# Install Gunicorn
pip install gunicorn

# Run Flask Server 1
cd backend/flask_server
gunicorn -w 4 -b 0.0.0.0:5000 --timeout 300 app:app

# Run Flask Server 2
cd backend/flask_server_2
gunicorn -w 4 -b 0.0.0.0:5001 --timeout 300 app:app
```

### Using Supervisor (Auto-restart)

```ini /etc/supervisor/conf.d/flask-server-1.conf
[program:flask-server-1]
command=gunicorn -w 4 -b 0.0.0.0:5000 --timeout 300 app:app
directory=/path/to/backend/flask_server
autostart=true
autorestart=true
stderr_logfile=/var/log/flask-server-1.err.log
stdout_logfile=/var/log/flask-server-1.out.log
```

### Using Docker

<CodeGroup>
```dockerfile Flask Server 1
FROM python:3.10-slim

WORKDIR /app

RUN apt-get update && apt-get install -y \
    libsm6 libxext6 libxrender-dev libgomp1 \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 5000
CMD ["gunicorn", "-w", "4", "-b", "0.0.0.0:5000", "--timeout", "300", "app:app"]
```

