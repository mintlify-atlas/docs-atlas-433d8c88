## Overview

The IPD community feed allows you to discover posts from other users, engage with content through likes and comments, and share your thoughts on deepfake analysis results.

## Accessing the Community Feed

Navigate to the community page:
- Visit `/community` from the navigation
- Or click the **"Share a Video"** button from the home page

<Info>
  The community feed is publicly accessible, but you must be authenticated to like or comment on posts.
</Info>

## Community Feed Interface

The feed displays posts in reverse chronological order (newest first):

### Feed Layout

<Steps>
  <Step title="Header Section">

  </Step>
  
  <Step title="Post Cards">

## Browsing Posts

### Viewing a Post

Click anywhere on a post card to:
- Navigate to the post details page
- View full content and media
- See all comments
- Access analysis results (for videos)

**Post URLs:**
- Post details: `/posts/{post_id}`
- Video analysis: `/posts/{post_id}/analysis`

## Liking Posts

Show appreciation for posts by liking them:

<Steps>
  <Step title="Click the Heart Icon">
    Navigate to the post details page and click the heart icon below the media.
  </Step>
  
  <Step title="Toggle Like Status">
    The API toggles your like:
    - First click: Adds your like
    - Second click: Removes your like
    
    ```javascript
    const likeIndex = post.likes.indexOf(userId);
    if (likeIndex === -1) {
      post.likes.push(userId); // Add like
    } else {
      post.likes.splice(likeIndex, 1); // Remove like
    }
    ```
  </Step>
  
  <Step title="Visual Feedback">
    The heart icon updates:
    - **Liked**: Red filled heart
    - **Not liked**: Gray outline heart
    - Like count updates in real-time
  </Step>
</Steps>

<Warning>
  You must be authenticated to like posts. Clicking the heart icon while logged out will redirect you to the login page.
</Warning>

### Like API Endpoint

```
POST /api/posts/{postId}/like
Authorization: Bearer <token>
```

Response:
```json
{
  "likes_count": 15
}
```

## Commenting on Posts

Engage in discussions by commenting on posts:

<Steps>
  <Step title="Navigate to Post Details">
    Click on a post to open the full details page with the comments section.
  </Step>
  
  <Step title="Write Your Comment">
    Find the comment form below the post:
    - Large textarea input
    - Placeholder: "Add a comment..."
    - Required field validation
    
    <Tip>
      Comments support multiple lines and whitespace. Press Enter for new lines.
    </Tip>
  </Step>
  
  <Step title="Submit Comment">
    Click the **Send** button (paper plane icon) to post your comment.
    
    The system will:
    1. Validate you're authenticated
    2. Send comment to the server
    3. Add comment to the post
    4. Display your comment immediately
    5. Clear the input field
  </Step>
  
  <Step title="View Comments">
    All comments appear below the form:
    - Newest comments first
    - Username and email displayed
    - Post date shown
    - Light gray background for each comment
  </Step>
</Steps>

### Comment Display

```jsx
<Comment>
  <Header>
    <Username>{comment.profiles.username}</Username>
    <Email>{comment.profiles.email}</Email>
    <Date>{formattedDate}</Date>
  </Header>
  <Content>{comment.content}</Content>
</Comment>
```

### Comment API

**Endpoint:**
```
POST /api/posts/{postId}/comments
Authorization: Bearer <token>
Content-Type: application/json

{
  "content": "Great analysis! This video looks authentic to me."
}
```

**Response:**
```json
{
  "id": "comment_123",
  "content": "Great analysis! This video looks authentic to me.",
  "created_at": "2026-03-01T10:30:00Z",
  "profiles": {
    "username": "user@example.com"
  }
}
```

<Info>
  Comments are stored with the post and persist across sessions. They're visible to all users.
</Info>

## Real-time Updates

The community feed includes smart polling for processing videos:

```javascript
useEffect(() => {
  const processingPosts = posts.some(
    post => post.media_type === 'video' && post.analysis_status === 'processing'
  );
  
  if (processingPosts && !pollInterval) {
    const interval = setInterval(async () => {
      const data = await api.getPosts();
      setPosts(data);
      
      // Stop polling when all analyses complete
      if (!data.some(post => post.analysis_status === 'processing')) {
        clearInterval(interval);
      }
    }, 5000); // Poll every 5 seconds
  }
}, [posts]);
```

<Info>
  Automatic polling stops when all videos finish processing to save resources.
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Posts Not Loading">
    If the feed shows no posts or loading errors:
    
    **Solutions:**
    - Check backend server is running (port 3000)
    - Verify database connection
    - Check browser console for errors
    - Try refreshing the page
    - Ensure you're not blocking API requests
  </Accordion>
  
  <Accordion title="Can't Like or Comment">
    If engagement features don't work:
    
    **Causes:**
    - Not authenticated
    - Token expired
    - Network issues
    
    **Solutions:**
    - Check you're logged in
    - Sign in again to refresh token
    - Verify network connectivity
    - Check browser console for 401 errors
  </Accordion>
  
  <Accordion title="Videos Not Playing">
    If videos fail to load or play:
    
    **Solutions:**
    - Check video file exists in /uploads
    - Verify backend server is running
    - Check browser console for 404 errors
    - Ensure video format is supported (MP4 recommended)
    - Try refreshing the page
  </Accordion>
  
  <Accordion title="Analysis Badge Not Updating">
    If processing badge doesn't change to results:
    
    **Causes:**
    - Analysis still in progress
    - Flask server not running
    - Analysis failed
    
    **Solutions:**
    - Wait longer (analysis can take 30-60 seconds)
    - Check Flask server status (port 5000)
    - Review backend logs for errors
    - Manually refresh the page
    - Check post analysis_status in database
  </Accordion>
  
  <Accordion title="Comments Not Appearing">
    If your comment doesn't show up:
    
    **Solutions:**
    - Refresh the page
    - Check that comment was saved (no error message)
    - Verify you're authenticated
    - Check network tab for failed requests
    - Ensure comment content is not empty
  </Accordion>
</AccordionGroup>
